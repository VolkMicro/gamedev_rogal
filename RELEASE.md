# Релиз «Фитиль» — чеклист

Игра задеплоена на https://volkmicro.github.io/gamedev_rogal/ (авто-деплой из
`main` через GitHub Actions). Бот: **@WickGameBot_bot**, menu button уже
указывает на Pages-URL. Ниже — что осталось сделать руками в BotFather
(нужен Telegram-аккаунт владельца бота), и что уже готово в репо.

## Действия в BotFather (5 минут)

1. `/setuserpic` → выбрать @WickGameBot_bot → отправить файл
   `public/icon.png` (512×512, пиксельная свеча — лежит в репо и на
   https://volkmicro.github.io/gamedev_rogal/icon.png).
2. `/setdescription` → текст:
   > Фитиль — мрачный рогалик с пиксельной физикой. Копай, замораживай,
   > поджигай и топи: каждая песчинка симулируется. Спустись через 4 биома
   > к Сердцу горы. Каждая смерть — навсегда, но эссенция остаётся.
3. `/setabouttext` → короткий вариант:
   > Noita-подобный рогалик прямо в Telegram. Падающий песок, живой огонь,
   > пермадет.
4. (опционально) `/setcommands`:
   ```
   play - Играть
   ```

## Аналитика — решение

В v1 аналитики НЕТ намеренно: у GitHub Pages нет серверной части, а
сторонние счётчики (GA и т.п.) в Telegram WebView — лишний вес и вопросы
приватности ради метрик, которые пока не на что влиять. Если понадобится:
самый лёгкий путь — Telegram-нативный `sendData`/бот-логика или
CloudStorage-счётчики, решение отложено до реальной потребности.

## Финальная проверка перед анонсом

1. Открыть бота на реальном телефоне → игра грузится, идёт в ландшафт.
2. Пройти первый этаж: копание песка, вода стекает, подсказки видны.
3. Умереть → сводка → «В лагерь» → купить заклинание → новый забег.
4. Проверить звук и вибрацию (и что мьют работает).
5. «Поделиться» из сводки — открывается шер-лист Telegram.
