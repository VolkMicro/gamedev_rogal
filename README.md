# «Фитиль» (Wick)

2D-рогалик с попиксельной falling-sand симуляцией (Noita-like), Telegram Mini
App. Полный дизайн — в [GDD.md](./GDD.md).

## Стек

TypeScript + Vite + PixiJS (собственный игровой цикл вокруг самописной
клеточной симуляции на typed arrays — без Phaser).

## Разработка

```bash
npm install
npm run dev       # http://localhost:5173
npm run build     # прод-сборка в dist/
```

## Статус

**Этап 1 (перф-гейт)** — голая симуляция (песок/вода/огонь) + бег/прыжки
персонажа. Критерий: 60 fps на среднем Android (30 fps — абсолютный минимум).

- Симуляция: чанки 64×64, «спящие» чанки без движения пропускаются,
  внутреннее разрешение 400×240, апскейл nearest-neighbor.
- Управление (тест-сборка): левая половина экрана — стик влево/вправо, тап
  по правой половине — прыжок. Клавиатура для десктоп-теста: стрелки/WASD +
  пробел.
- В левом верхнем углу — счётчик FPS и активных чанков, в правом верхнем —
  кнопка "Stress test" (форсирует больше материала в симуляции для проверки
  худшего случая).

## Как проверить сборку на телефоне через Telegram

1. **Задеплоить `dist/`** — при пуше в `main` GitHub Actions
   (`.github/workflows/deploy.yml`) сам собирает и публикует на GitHub Pages
   по адресу `https://<твой-github-юзер>.github.io/gamedev_rogal/`. Проверить
   статус: вкладка **Actions** в репозитории на GitHub.
2. **Создать тестового бота** (одноразово, через Telegram):
   - Написать [@BotFather](https://t.me/BotFather) → `/newbot`, задать имя.
   - `/newapp` (или `/setmenubutton` / `/mybots` → выбрать бота → **Bot
     Settings → Menu Button** для более старых версий BotFather) → указать
     тот же URL с GitHub Pages.
   - BotFather выдаст ссылку вида `https://t.me/<имя_бота>/<имя_app>`.
3. **Открыть эту ссылку в Telegram на телефоне** — Mini App откроется прямо
   в клиенте. Смотреть на счётчик FPS в углу: держать открытым 20-30 секунд,
   походить (стик слева), попрыгать (тап справа), включить "Stress test" —
   FPS не должен уходить ниже 30 (цель — стабильные 60).
4. Если FPS ниже 30 на среднем Android — по ТЗ переходим к порту ядра
   симуляции на Rust/WASM до дальнейшей работы над геймплеем.
